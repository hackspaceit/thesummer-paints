A) Server Requirements - top

Please make sure that the following server requirements are met as listed:

  • PHP version 5.5 or newer is recommended. It should work on 5.4.8 as well, but we strongly advise you NOT to run such old versions of PHP, because of potential security and performance issues, as well as missing features.
  • URL rewrite enabled (usually enabled by default on most servers)
  • MySQL 5.1.0 or above
  • allow_url_fopen must be enabled in php.ini (if not enabled)

B) Installation Guide - top

Please make sure that you meet the server requirements before starting installation.

After purchasing CryptoNet PHP Script from Codecanyon, Download the complete zip package of our PHP script & extract it. It contains documentation folder and CryptoNet.zip files.



STEP 1: Upload the CryptoNet.zip folder to your server and uncompress/extract all the files to the root directory of your domain name on your web server. Normally the index.php file will be at your root. You can upload the package on your web server in two ways via FTP or File manager.



STEP 2: Create a MySQL database and database user on your server who has all the permissions to access and modify database.

If you do not know, how to create a database then ask for help to your hosting provider or search help topics on your hosting or search on any search engine for How to create a MYSQL database. You will found so many step by step guides and video tutorials.

STEP 3: Run the auto installer script by accessing the URL https://www.your-domain-name.com/install/ (If you have uploaded the files in the root directory of your domain) or https://www.your-domain-name.com/subdirectory/install/ (If you have uploaded the files in the subdirectory of your domain) in a web browser.

Note: If you want to install the Corona Net PHP Script on Non-SSL Version of your domain name then firstly open .htaccess file in the root directory of your server and comment out the code in line 155 and 156 which is as follows:

Before Comment Out:

RewriteCond %{HTTPS} off
RewriteRule (.*) https://%{HTTP_HOST}%{REQUEST_URI} [R,L]

After Comment Out:

#RewriteCond %{HTTPS} off
#RewriteRule (.*) https://%{HTTP_HOST}%{REQUEST_URI} [R,L]

After that, please use http instead of https in STEP 3.

STEP 4: Enter the database details which you have created on your server in the STEP 2 and click “Install” button.

STEP 5: On the next screen, Enter email and password to access admin area. Now, again Click “Install” button.

Congratulations, Your PHP Script is installed successfully now and you will redirected to your fully automated Cryptocurrency market capitalization website.

You can access the admin area by visiting this URL https://www.your-domain-name.com/admin/

E) GDPR Compliance - top

We try our best to make CryptoNet PHP script GDPR Compliance. There is a consent to the user in the contact us page form. The user must accept consent before submitting the form and all the contact entries you can see from the backend. As per GDPR, if a user want to request or delete personal information then you can easily access or delete the contact form queries from the admin area.

Also, there is a cookie consent bar will be shown to the user in the header bar to accept privacy policy. You must create your own privacy policy as per GDPR. For the data request and removal process, you must mention your email address for this in your privacy policy.

If you want to show Google Adsense ads, then you can setup the non-personalised ads for the EU users from the Google Adsense dashboard.

If you want to add the Google Analytics code to the website admin area, then you must mention this as essential cookies in your privacy policy.

H) How to Update - top

Important Note: Before you get started, it’s a good idea to back up your website. This means if there are any issues you can restore your website.

How to update Crypto Net to version 5.4

  • Download Crypto Net from CodeCanyon and Unpack archive
  • Open "CryptoNet.zip"
  • Extract the “CryptoNet” directory to your computer
  • Connect to your FTP server
  • Upload the “application->controllers” & “application->core” & “application->views” directory to the “application->controllers” & “application->core” & “application->views” and rewrite all original files
  • Upload the “assets” directory to the “/assets” and rewrite all original files
  • Create the folder named "json" in the “upload” directory if you are updating from version 4.1 or below.
  • Done
